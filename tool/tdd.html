# TDD

테스트 주도 개발(Test-driven development TDD)은 매우 짧은 개발 사이클을 반복하는 소프트웨어 개발 프로세스 중 하나이다.
개발자는 먼저 요구사항을 검증하는 자동화된 테스트 케이스를 작성한다.(테스트 코드를 먼저 작성)
그런 후에, 그 테스트 케이스를 통과하기 위한 최소한의 코드를 생성한다. 마지막으로 작성한 코드를 표준에 맞도록 리팩토링한다.

테스트 코드 자체가 좋은 문서화가 된다.
main repository에 merge할 때 test코드를 포함해서 한다.
일반적으로 UI 코드는 TDD 하지 않는다.

# Test Pyramid

  • E2E(end to end) Test : 사용flow를 test
  • Integration Test : 상호작용 통합 test
  • Unit Test : 단위(함수, 모듈, 클래스) test
